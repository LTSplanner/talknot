# 弊社ナレッジを共有ドライブに永続保存する設定手順

専用AIの頭脳（社内ルール・商品知識・営業トーク）を、アプリの再起動でも消えないよう
**Google スプレッドシート**に保存します。人もシートを直接開いて閲覧・編集できます。

この設定をするまでは、ナレッジはアプリ内のローカル保存（再起動で消える可能性あり）で動きます。
設定が完了すると自動でスプレッドシート保存に切り替わります。

---

## 全体像

```
[TalKnot アプリ] ──読み書き──> [共有ドライブのスプレッドシート] <──人も閲覧・編集
                （最小権限の専用アカウントで接続。録画には一切触れない）
```

---

## 手順（管理者・一度だけ／15分ほど）

### ① 保存用スプレッドシートを作る
1. **共有ドライブ**（個人マイドライブではなく、チームの共有ドライブ推奨）に
   新しい Google スプレッドシートを作成。名前は例：`TalKnot 弊社ナレッジ`
2. 1枚目のシート名（タブ名）を **`Knowledge`** にする
3. 1行目に見出しを入れる：`A1=category` / `B1=point` / `C1=added_at`
4. URL の中の **スプレッドシートID** を控える
   `https://docs.google.com/spreadsheets/d/`**`ここがID`**`/edit`

### ② 書き込み用の「最小権限アカウント」を作る
> 録画を読める強力な既存アカウントは**使いません**。万一漏れても被害をこのシート1枚に限定するためです。

1. Google Cloud Console → 対象プロジェクト → **IAMと管理 → サービスアカウント**
2. **サービスアカウントを作成**（例：`talknot-knowledge`）。役割は付けなくてOK
3. 作成後、その行 → **キー → 鍵を追加 → JSON** をダウンロード（このファイルが鍵）
4. **Google Sheets API** を有効化：APIとサービス → ライブラリ → 「Google Sheets API」→ 有効にする
5. 作成したサービスアカウントの**メールアドレス**（`...@....iam.gserviceaccount.com`）を控える

### ③ シートを専用アカウントに共有する
1. ①のスプレッドシートを開き **共有**
2. ②で控えた**サービスアカウントのメール**を追加し、権限を **編集者** にする

### ④ アプリに設定を登録する（Streamlit Cloud の Secrets）
Streamlit Cloud → アプリ → **Settings → Secrets** に以下を追加（TOML 形式）：

```toml
KNOWLEDGE_SHEET_ID = "①で控えたスプレッドシートID"

# ②でダウンロードした JSON 鍵の中身をそのまま貼り付ける（三重引用符で囲む）
KNOWLEDGE_SA_JSON = '''
{
  "type": "service_account",
  "project_id": "...",
  "private_key": "-----BEGIN PRIVATE KEY-----\n...\n-----END PRIVATE KEY-----\n",
  "client_email": "talknot-knowledge@....iam.gserviceaccount.com",
  ...
}
'''
```

保存するとアプリが自動再起動し、以降ナレッジはスプレッドシートに保存されます。

---

## 動作確認
1. アプリで商談を1本評価する
2. スプレッドシートを開く → `Knowledge` シートに行が増えていれば成功
3. アプリの「🧠 弊社ナレッジ」タブにも同じ内容が表示される

---

## ローカル開発で使う場合（任意）
`.env` に次を設定すれば、ローカルでもスプレッドシート保存を使えます：

```
KNOWLEDGE_SHEET_ID=スプレッドシートID
KNOWLEDGE_SA_FILE=secrets/talknot-knowledge-sa.json
```

---

## 補足・注意
- **このアカウントは録画ドライブには一切アクセスできません**（共有したシート1枚だけ）。
  以前ご懸念のあった「全員の録画を読める鍵をクラウドに置く」問題は起きません。
- 人がスプレッドシートを直接編集して、誤った知識を消したり正しい知識を足したりできます。
  アプリは次回の評価時にその内容を読み込みます。
- 知識は上限 150 件で、古いものから自動的に間引かれます（容量とAIコストを一定に保つため）。
